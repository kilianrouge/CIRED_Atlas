Import Conditions from JSON

Paste a JSON array of condition objects. Each condition needs a type and usually a value. You can ask an AI to generate this — share the schema below.

Show condition schema / AI prompt 
Generate a JSON array of search conditions for the ATLAS literature discovery tool.
Each condition is an object with these fields:

  type  (required) — one of:
    "keywords_title_abstract"  full-text in title + abstract; value = query string (AND/OR/"phrases")
    "keywords_title"           full-text in title only
    "author"                   OpenAlex author ID (A…); _display = human name
    "journal"                  OpenAlex source ID (S…); _display = journal name
    "field"                    OpenAlex field ID; value = one of:
                                 fields/20  Economics, Econometrics and Finance
                                 fields/14  Business, Management and Accounting
                                 fields/33  Social Sciences
                                 fields/23  Environmental Science
                                 fields/22  Engineering
                                 fields/17  Computer Science
                                 fields/02  Arts and Humanities
                                 fields/32  Political Science
                                 fields/12  Agricultural and Biological Sciences
                                 fields/19  Earth and Planetary Sciences
    "domain"                   OpenAlex domain; value = one of:
                                 domains/1  Physical Sciences
                                 domains/2  Social Sciences
                                 domains/3  Life Sciences
                                 domains/4  Health Sciences
    "citing_library"           papers citing my Zotero collection (no value needed)
                                 field / domain keys accepted for scoping, e.g. "field":"fields/20"
    "author_in_library"        prolific authors in my collection; min_papers (int, default 3)
                                 field / domain keys accepted for scoping

  value      (string) — query text or entity ID
  label      (string, optional) — human label shown as a chip
  _display   (string) — human-readable name for author/journal IDs
  exclude_title  (array of strings, optional) — words that must NOT appear in title

Example:
[
  {"type":"keywords_title_abstract","value":"degrowth OR \"post-growth\"","label":"Degrowth"},
  {"type":"field","value":"fields/23","label":"Environmental Science"},
  {"type":"citing_library"}
]
